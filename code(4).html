üë®‚Äçüè´ Teaching Notes - File System

üéØ Core Concepts to Teach:

  • NTFS Timestamps: Created, Modified, Accessed, MFT Modified
  • Hash Values: Digital fingerprints for file identification
  • Prefetch Files: Windows performance optimization that helps forensics
  • File Allocation: How deleted files can be recovered

üó£Ô∏è Explanation Techniques:

For Hashes: "Think of MD5/SHA256 like a fingerprint. Even changing one bit changes the entire hash. It's how we prove file integrity."

For Prefetch: "Windows caches program startup info for speed. Ironically, this helps us track exactly what malware did."

For Timestamps: "NTFS tracks four timestamps. Show how malware tries to hide by matching legitimate file times."

üîç Hands-On Activities:

  • Calculate MD5 hash of a file using certutil or PowerShell
  • Examine prefetch files in hex editor
  • Show NTFS timestamp manipulation techniques
  • Demonstrate file carving from unallocated space

‚ö†Ô∏è Important Caveats:

  • SSD TRIM commands can eliminate file recovery opportunities
  • Encrypted filesystems require additional considerations
  • Cloud storage sync can complicate timeline analysis
  • Modern malware often uses fileless techniques

üë®‚Äçüè´ Teaching Notes - Malware Analysis

üó£Ô∏è Courtroom Testimony Tips:

‚ö†Ô∏è Common Legal Pitfalls:

  • Missing documentation breaks chain of custody
  • Unauthorized access can violate privacy laws
  • Exceeding search warrant scope invalidates evidence
